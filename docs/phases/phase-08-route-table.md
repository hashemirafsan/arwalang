# Phase 08 - Route Table Builder & Validator

## Objective

Build a static route table from controller and HTTP method annotations, then validate route conflicts and path-parameter bindings.

## Delivered Scope

- Implemented `HttpMethod`, `Route`, and `RouteTable` in `src/routes/registry.rs`.
- Implemented `RouteTableBuilder::build` to collect controller-prefixed routes from AST classes/methods.
- Implemented route path normalization and prefix/method path joining.
- Implemented path parameter extraction from `:param` segments.
- Added duplicate route validation (`ROUTE001`).
- Added path parameter binding validation for:
  - unbound path parameters (`ROUTE002`)
  - unused `#[param]` annotations (`ROUTE003`)
- Added unit tests for all supported HTTP methods and multi-controller scenarios.

## Key Implementation Details

- Effective route path is generated by canonical normalization to avoid false negatives on leading/trailing slash differences.
- Validation cross-links route handlers back to original method parameter annotations for compile-time consistency checks.
- Duplicate route detection keys routes by `(HttpMethod, full_path)`.
